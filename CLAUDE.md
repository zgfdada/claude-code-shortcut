# Claude Code 项目规范

## 项目文件夹结构规范

根目录/
├── README-CODE.md # 面向开发者：技术说明、架构、部署流程
├── README-USER.md # 面向使用者：安装、使用、功能说明
├── .gitignore # Git 忽略文件
│
├── code/ # 源代码目录（开发用），永远不要在用户文件夹存储数据，数据都存在可执行程序同一目录下的data文件夹中，项目编译时保留之前的data文件夹
├──────build/ # 构建相关文件：vite配置、tailwind配置、构建脚本、构建输出
├──────resources/ # 静态资源文件：图标、图片等
├──────Test/ # 项目代码做单元测试、脚本测试等测试相关的文件位置
└── release/ # 编译后生成的程序、可执行文件、发布包

## 对于bug修复即功能优化或需求新增

1. 先不对项目代码进行改动，优先完成功能点的单元测试验证，通过后才进行项目代码的改动

2. 添加足够多足够细节的日志，方便bug溯源

## 运行过程中要求

1. 从头到尾保持使用中文与我交流

2. 执行任何终端命令前都先告诉我执行命令的意义
