# Claude Code Shortcut 使用指南

## 简介

Claude Code Shortcut（打包产物名称为“zgf的命令行辅助小工具”）是一款基于 Electron + React + TypeScript 的桌面 GUI 工具，用于集中管理常用命令、快速搜索、复制，并在 **Windows** 上将命令直接发送到已绑定的终端窗口。

本次版本已补齐多平台桌面打包能力：

- **Windows**：完整支持命令管理 + 终端绑定 / 发送 / 跟随
- **Linux**：支持 GUI、命令管理、搜索、收藏、分类、主题等功能；**终端绑定相关功能默认禁用**
- **macOS**：已提供打包产物；功能定位与 Linux 相同，**终端绑定相关功能默认禁用**

## 平台支持说明

| 平台 | GUI / 命令管理 | 绑定终端 | 发送到终端 | 跟随终端 |
|------|----------------|----------|------------|----------|
| Windows 10/11 | 支持 | 支持 | 支持 | 支持 |
| Linux | 支持 | 不支持 | 不支持 | 不支持 |
| macOS | 支持 | 不支持 | 不支持 | 不支持 |

说明：Linux 和 macOS 上会直接禁用相关按钮，不会因为缺少 Windows API 而导致程序启动失败。

## 主要功能

- **悬浮置顶窗口**：随时呼出常用命令
- **全局热键唤起**：默认 `Ctrl+Shift+Space`
- **智能搜索**：支持命令内容和中文描述搜索
- **命令收藏**：常用命令置顶显示
- **分类管理**：支持分类筛选与分类颜色设置
- **主题设置**：可自定义背景色、文字色、强调色和窗口大小
- **终端绑定（仅 Windows）**：将命令发送到指定 CMD / PowerShell / Windows Terminal 窗口
- **终端跟随（仅 Windows）**：小工具跟随已绑定终端显示、移动与前后台状态
- **使用统计**：自动记录使用次数，便于优先展示常用命令

## 安装与启动

## Windows

### 安装版

直接运行安装包：

- `zgf-cmd-helper-Setup-1.0.1.exe`

### 免安装版

直接运行：

- `win-unpacked/zgf的命令行辅助小工具.exe`

## Linux

### AppImage

先赋予执行权限，再运行：

```bash
chmod +x "zgf的命令行辅助小工具-1.0.1.AppImage"
./zgf的命令行辅助小工具-1.0.1.AppImage
```

如果提示缺少 `libfuse.so.2`，请先安装 FUSE 兼容库：

- Ubuntu 24.04 / Debian 新版本常见包名：`libfuse2t64`
- 部分发行版包名可能为：`libfuse2`

### 免安装目录

也可以直接运行 `linux-unpacked/` 中的主程序。

如果遇到 `chrome-sandbox` 权限问题，有两种做法：

1. 临时运行：

```bash
./linux-unpacked/claude-code-shortcut-code --no-sandbox
```

2. 正确配置 `chrome-sandbox` 的所有者和 SUID 位后再运行。

## macOS

解压以下文件后运行 `.app`：

- `zgf的命令行辅助小工具-1.0.1-mac.zip`

如果系统提示应用未签名，可使用“右键打开”或在系统安全设置中允许启动。

## 基本操作

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Shift+Space` | 唤起 / 隐藏窗口 |
| `Esc` | 隐藏窗口到托盘 |
| `Ctrl+X` | 完全退出程序 |

> 非 Windows 平台的系统快捷键/托盘行为取决于当前桌面环境与系统权限；若全局热键未生效，仍可直接通过应用窗口使用全部命令管理功能。

## 使用说明

### 添加命令

1. 点击搜索框右侧的 **+** 按钮
2. 输入命令内容，例如 `git status`
3. 输入中文描述，例如 `查看 git 状态`
4. 可选：输入分类，例如 `git`、`cmd`、`Claude`
5. 点击保存

### 命令管理

- **复制**：点击命令右侧的 **复制** 按钮
- **编辑**：点击 **编辑** 按钮修改命令内容
- **删除**：点击 **删除** 按钮移除命令
- **收藏**：点击 ☆ 标记为收藏，收藏命令会优先显示

### 分类筛选

- 点击顶部分类标签快速筛选
- 点击分类旁的小圆点可修改分类颜色

### 主题设置

1. 点击搜索框右侧的 **太阳图标**
2. 调整背景色、文字色、强调色
3. 调整窗口大小
4. 点击 **重置为默认主题** 恢复默认设置

### 绑定终端（仅 Windows）

1. 点击顶部 **绑定终端** 按钮
2. 选择要绑定的 CMD / PowerShell / Windows Terminal 窗口
3. 绑定成功后，命令列表会显示 **发送** 按钮
4. 点击发送即可把命令注入到目标终端
5. 若终端窗口关闭，程序会自动检测并解绑，并提示“终端已关闭”

### 跟随终端（仅 Windows）

绑定终端后可开启 **跟随**：

- 小工具会贴在终端旁边
- 终端最小化时，小工具会同步隐藏
- 终端恢复后，小工具会同步显示
- 终端前后台切换时，小工具会同步调整置顶状态

## 数据存储与备份

- 程序使用 `data.db` 保存命令、收藏、分类和主题设置
- 程序运行日志保存在 `app.log`
- 请将程序放在**可写目录**下运行
- 备份时请一并保存 `data.db`
- 升级版本时，如需保留原有数据，请将旧版本中的 `data.db` 复制到新版本对应目录

## 常见问题

### 为什么 Linux / macOS 上“绑定终端”按钮是灰色的？

这是当前版本的设计行为。终端绑定、发送和跟随功能依赖 Windows API，目前仅在 Windows 上实现；Linux / macOS 会保留 GUI 与命令管理功能，但禁用终端相关按钮。

### Linux 上 AppImage 无法启动，并提示 `libfuse.so.2` 缺失怎么办？

安装 `libfuse2t64` 或 `libfuse2` 后重试。

### Linux 上运行 `linux-unpacked` 时提示 `chrome-sandbox` 配置错误怎么办？

可以先使用 `--no-sandbox` 临时运行，或按系统要求为 `chrome-sandbox` 配置 root 所有者和 `4755` 权限。

### 为什么发送命令后终端没有反应？

请确认：

- 当前平台是 **Windows**
- 已正确绑定目标终端
- 目标终端窗口处于可接收输入状态
- 终端没有被关闭或卡死

### 如何备份我的命令？

直接备份程序目录中的 `data.db` 文件即可。

### 程序无法启动怎么办？

可优先检查：

- 是否缺少运行依赖（Linux 常见为 FUSE）
- 是否被系统安全策略或杀毒软件拦截
- 是否存在 `app.log`，可将其提供出来用于排查
